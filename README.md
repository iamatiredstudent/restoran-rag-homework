# RAG-система по базе знаний для управления запасами и закупками в ресторанах

## Описание проекта

Проект представляет собой учебную RAG-систему для поиска ответов по текстовой базе знаний о ресторанной аналитике. Система индексирует документы о прогнозировании спроса, управлении запасами, закупках, списаниях, поставщиках, food cost, ABC/XYZ-анализе, инвентаризации и контроле качества данных.

Цель проекта — показать полный pipeline RAG-приложения: подготовка данных, ingestion, нарезка документов на чанки, построение TF-IDF индекса, retrieval, формирование demo-ответа и отображение результата в Streamlit-интерфейсе.

Система не использует внешнюю LLM. Ответ формируется только на основании найденных фрагментов. Если релевантный контекст не найден, система должна отказаться отвечать и не выдумывать информацию.

## Функциональность

Проект реализует следующие функции:

* загрузка исходных текстовых данных из `data/raw/datasets.json`;
* преобразование исходных записей в документы;
* нарезка документов на чанки;
* построение TF-IDF индекса;
* поиск релевантных фрагментов по вопросу пользователя;
* формирование demo-ответа только на основании найденных фрагментов;
* отображение ответа, найденных фрагментов, источников и score в Streamlit UI;
* отказ от ответа, если релевантных фрагментов в базе знаний нет.

## Данные

В проекте используется учебный синтетический датасет по теме управления запасами и закупками в ресторанах.

Файл с исходными данными:

```text
data/raw/datasets.json
```

Количество записей:

```text
1200
```

Каждая запись содержит:

```text
id — идентификатор записи
name — название кейса
text — текстовое описание управленческой ситуации
```

Индексируется поле:

```text
text
```

Поля `id` и `name` используются как метаданные для отображения источников.

Тематика записей включает:

* прогнозирование спроса;
* управление складскими остатками;
* оптимизацию закупок;
* контроль списаний;
* ABC-анализ ингредиентов;
* XYZ-анализ стабильности потребления;
* расчет food cost;
* управление поставщиками;
* инвентаризацию;
* сезонность спроса;
* планирование меню;
* контроль дефицита;
* аналитику продаж;
* управление закупочными ценами;
* анализ оборачиваемости запасов;
* контроль качества данных;
* выявление аномалий;
* планирование банкетов;
* учет скоропортящихся продуктов;
* контроль маржинальности блюд.

Данные создаются скриптом:

```bash
uv run python scripts/generate_datasets.py
```

Скрипт генерирует 1200 текстовых записей в формате JSON. Датасет является учебным и не содержит персональных данных, коммерческой тайны, реальных поставщиков, реальных ресторанов или закрытой финансовой информации.

## Pipeline

Проект реализует следующий pipeline:

```text
data/raw/datasets.json
→ data/processed/documents.jsonl
→ data/processed/chunks.jsonl
→ data/index/chunks.jsonl
→ data/index/vectorizer.pkl
→ data/index/matrix.npz
→ retrieval
→ demo-answer
→ Streamlit UI
```

Этапы pipeline:

1. `scripts/generate_datasets.py` создает исходный датасет.
2. `scripts/ingest.py` преобразует записи из `datasets.json` в документы.
3. `app/chunker.py` нарезает документы на чанки.
4. `scripts/build_index.py` строит TF-IDF индекс.
5. `app/retriever.py` ищет релевантные чанки по запросу пользователя.
6. `app/generator.py` формирует demo-ответ только по найденным фрагментам.
7. `app/main.py` отображает интерфейс в Streamlit.

## Demo-вопросы

### Вопрос 1

```text
Как ресторану снизить списания скоропортящихся продуктов?
```

Ожидаемый результат: система должна найти документы про контроль списаний, скоропортящиеся продукты, прогноз спроса и управление запасами. Ответ должен быть сформирован только на основании найденных фрагментов.

### Вопрос 2

```text
Какие метрики нужно использовать для контроля закупок в ресторане?
```

Ожидаемый результат: система должна найти фрагменты про food cost, оборачиваемость запасов, уровень списаний, точность прогноза, маржинальность блюд и уровень сервиса поставщика.

### Вопрос 3

```text
Зачем ресторану нужен ABC-анализ ингредиентов?
```

Ожидаемый результат: система должна объяснить, что ABC-анализ помогает выделить наиболее значимые продуктовые категории, требующие регулярного контроля, поскольку ошибки в закупках, хранении или списаниях таких ингредиентов сильнее влияют на экономику ресторана.

## Negative-вопрос

```text
Кто проживает на дне океана?
```

Ожидаемое поведение: система должна отказаться отвечать, потому что в базе знаний нет релевантного контекста по этому вопросу.

Ожидаемый ответ:

```text
В базе не найдено релевантных фрагментов. Ответить по данным невозможно.
```

## Проверка тестов

Запуск всех тестов:

```bash
uv run pytest tests/ -v
```

Ожидаемый результат:

```text

```

В проекте проверяются:

* корректная нарезка текста на чанки;
* ограничение размера чанков;
* сохранение `doc_id`;
* создание `chunks.jsonl`;
* возврат top-k результатов retrieval;
* наличие `doc_id` и `score` в результатах поиска;
* обработка пустого запроса;
* наличие источников в ответе;
* отказ от ответа при отсутствии релевантного контекста.

## Ограничения

Это учебная RAG-система. Она использует TF-IDF и cosine similarity, поэтому качество поиска зависит от совпадения терминов в вопросе и корпусе документов.

Система не должна использовать информацию вне базы знаний. Если релевантные фрагменты не найдены, она должна вернуть отказ от ответа.

Датасет является синтетическим и используется только для демонстрации pipeline. Он не предназначен для принятия реальных управленческих решений без проверки на фактических данных ресторана.

## Критерии готовности

Проект считается готовым, если:

* `data/raw/datasets.json` содержит 1200 записей;
* индекс успешно собирается;
* Streamlit-приложение запускается;
* demo-вопросы возвращают ответы по найденным фрагментам;
* negative-вопрос возвращает отказ;
* найденные фрагменты отображаются с `doc_id`, `score`, `name` и `text`;
* тесты проходят успешно;
* данные и ограничения описаны в `doc/DATA.md`;
* README содержит команды запуска и demo-вопросы.
